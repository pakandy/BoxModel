Cascading Style Sheets (CSS)

CSS3 Values and Units Module

This CSS module describes the common values and units that CSS properties accept and the syntax used for describing them in CSS property definitions.

Read more— W3C specifications

CSS values

CSS values are set against CSS Properties and reside within CSS declaration block. The value definition for each CSS property can be different depending on the type of the CSS property.

CSS values can be keywords such as auto, collapse, initial, etc.; data types such as string, numeric, percentage, url, etc.

For example:

  • color: initial / inherit / <color, such as red>;
  • width: auto / initial / inherit / <numeric, such as 150px> / <percentage, such as 50%>;
  • border-collapse: separate / collapse / initial / inherit;

CSS units

For numeric data type there are also different units. We'll look at some units used to define CSS sizes for dimension and distance.

pt versus px versus em versus rem verse percentage (%)

  • Points (pt): Points are traditionally used in print media (anything that is printed on paper, etc.). One point is equal to 1/72 of an inch. Points are fixed-size units and cannot scale in size.
  • Pixels (px): Pixels are, similar to Points, fixed-size units that are used in screen media (i.e. to be read on the computer screen). One pixel is equal to one dot on the computer screen (the smallest division of your screen’s resolution). One problem with the pixel unit is that it does not scale. The elements defined using pixels will look different on another device with lower or higher resolution.
  • "Ems" (em): An em is equal to the font-size of the element (or the font-size of its nearest parent element), for instance, if the font-size of the element is 16px, 1em is equal to 16px. Ems are scalable in nature, so 2em would equal 32px, .5em would equal 8px, etc. So, the elements will look the same on different devices regardless of their resolution.
  • "Rems" (rem): An rem is equal to the font-size of the root element, for instance, if the font-size of the root element is 16px, 1rem is equal to 16px. Rems are scalable in nature, so 2rem would equal 32px, .5rem would equal 8px, etc. So, the elements will look the same on different devices regardless of their resolution.
  • Percent (%): The percentage, similiar to the Em and Rem, is scalable. If you set an element's font size as a percentage it will be a percentage of the font size of the parent element.
